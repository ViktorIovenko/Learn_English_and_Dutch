## Важные задачи

### 1. Критический баг: приложение вылетает при нажатии на `Настройки`

**Приоритет:** высокий, исправить в первую очередь.

**Проблема:**

В Android-приложении при нажатии на ссылку/кнопку `Настройки` приложение вылетает.

**Что нужно сделать:**

- Найти причину вылета при переходе в `Настройки`.
- Проверить обработчик нажатия на `Настройки`.
- Проверить экран/Activity/Fragment/Route настроек.
- Исправить ошибку, чтобы экран настроек открывался без падения приложения.
- После исправления проверить переход несколько раз подряд.

**Ожидаемое поведение:**

- Пользователь нажимает `Настройки`.
- Приложение не закрывается и не вылетает.
- Открывается экран настроек.
- Возврат назад из настроек работает нормально.

### Главная страница: визуально показывать, какие уроки нужно повторять

**Что нужно добавить:**

На главной странице со списком уроков нужно визуально показывать, какие уроки пользователь знает хорошо, а какие нужно повторить.

**Важно:**

Не нужно превращать страницу в «радугу». Не должно быть ярких разноцветных карточек, которые перегружают интерфейс.

**Ожидаемое поведение:**

- Карточки уроков сохраняют спокойный и чистый внешний вид.
- Добавляется небольшой визуальный элемент, который показывает состояние урока.
- Пользователь должен одним взглядом понимать, какой урок стоит повторить.

**Возможные варианты:**

- тонкая вертикальная полоска сбоку карточки;
- маленький индикатор или точка;
- небольшой прогресс-элемент;
- аккуратная шкала;
- лёгкий акцент без перегрузки интерфейса.

**Пример логики:**

- низкий процент знаний → небольшой акцент, что урок требует повторения;
- средний процент → нейтральное состояние;
- высокий процент → минимальный индикатор хорошего результата.

**Идея UX:**

Пользователь должен без чтения цифр быстро понимать, куда стоит зайти в первую очередь. Но интерфейс должен оставаться спокойным, чистым и не перегруженным цветами.
